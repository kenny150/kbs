## diretório de aramzenamento das configurações personalizadas de interfaces de rede /etc/sysconfig/network-scripts/
## Exemplo de um arquivo de configuração de placa de rede:

HWADDR=00:40:A7:21:12:85                            //Endereço MAC da placa
TYPE=Ethernet                                       //Tipo de conexão/placa
BOOTPROTO=dhcp                                      //Obterá ip via DHCP
DEFROUTE=yes
IPV4_FAILURE_FATAL=no
IPV6INIT=yes
IPV6_AUTOCONF=yes
IPV6_DEFROUTE=yes
IPV6_FAILURE_FATAL=no
NAME=enp6s0f0                                       //Nome do dispositivo
UUID=0e2a3030-70fb-4ec8-9d04-30fb6048912b           //ID da placa
ONBOOT=no                                           //Inicia automaticamente ao carregar o SO 
PEERDNS=yes                                         //Permite que o DHCP substitua informações do arquivo resolv.conf.
PEERROUTES=yes
IPV6_PEERDNS=yes
IPV6_PEERROUTES=yes

## Exemplo de configuração de placa de rede estatica:

DEVICE=eth1                                         //Nome do dispositvo
HWADDR=00:40:A7:21:12:85                            //Endereço MAC da placa
TYPE=Ethernet                                       //Tipo de conexão/placa
BOOTPROTO=none                                      //IP estatico
ONBOOT=yes                                          //Inicia automaticamente ao carregar o SO 
PEERDNS=yes                                         //Permite que o DHCP substitua informações do arquivo resolv.conf.
NM_CONTROLLED=no                                    //Não permite que o Network Manager controle a placa.
USERCTL=no                                          //Impede que usuários regulares sejam capazes de iniciar a placa de rede
IPADDR=192.168.0.140                                //ip da placa
NETMASK=255.255.255.0                               //Mascara de subrede
GATEWAY=192.168.0.1                                 //Gateway da placa


## Arquivo de cofigurações de rede /etc/sysconfig/network, o hostname pode ser configurado neste arquivo.
## Exemplo de configuração de um arquivo /etc/sysconfig/network

NETWORKING=yes
HOSTNAME=abc.exemplo.com
GATEWAY=192.168.0.1

## Arquivo /etc/resolv.conf recebe os valores de DNS
## Exemplo de cofiguração do resolv.conf

# Generated by NetworkManager
nameserver 200.204.0.10
nameserver 200.204.0.138
nameserver 192.168.0.1
 
NOTA: tabém é possível acrescentar em uma linha "search  exemplo.com    exemplo.org     exemplo.net" os endereços de pesquisa devem ser separadas por tabulações ou espaços.

## Arquivo de configuração /etc/nsswitch.conf permite configurar a ordem em que os nomes são resolvidos. 
## Exemplo de configuração do nssitch.conf.

hosts: 			files	dns               


NOTA: Essa configuração diz para o sistema que ele deve resolver um hostname primeiro procurando no arquivo /etc/hosts e então nos dns apontados no arquivo /etc/resolv.conf. É possível alterar a ordem da busca, assim como acrescentar o nis (network information service) ou nisplus.

## Para criar uma placa de rede secundária com o mesmo ip, no caso de um servidor web possuir vários dominios (por exemplo), apenas precisa copiar o arquivo da placa que fica dentro de /etc/syscnfig/network-scripts/ifcfg-eth0 e criar uma com o nome de ifcfg-et0:0. A placa secundaria deve ser editada, o campo ONBOOT deve ser substituido por ONPARENT, isso diz que ela deve ser ativada se a placa pai (eth0) for ativada. Com isso uma mesma placa de rede pode ouvir vários IP's.
exempo:

DEVICE=eth0:0
ONPARENT=yes
IPADDR=192.168.0.100
NETMASK=255.255.255.0 

DEVICE=eth0:1
ONPARENT=yes
IPADDR=192.168.0.101
NETMASK=255.255.255.0


## A agregação da placa de rede é utilizada para alta disponibilidade e desempenho, basicamente ela é uma segunda placa que possui o mesmo ip da primeira, assim que a placa principal fica indisponível, ela assume. Também pode ser usava para aumento do desempenho, por exemplo; se a placa principal estiver trabalhando para um envio grande de pacotes, as outras placas podem ser usadas para ajudar no envio assim melhorando o desempenho .
## Exemplo de como configurar uma placa de rede agregada.

- Criar um arquivo com o nome da placa dentro de /etc/sysconfig/network-scripts . 
touch /etc/sysconfig/network-scripts/ifcfg-placa-backup
- A configuração pode ser a do exemplo abaixo.

DEVICE=placa-backup
ONBOOT=yes
IPADDR=192.168.0.100
NETMASK=255.255.255.0
BOOTPROTO=none
BONDING_OPTS="mode=active-backup" //esse modo diz que a placa só assume quando a principal falhar.

- Em seguida editar o arquivo de configuração da placa principal
vi /etc/syconfig/network-scripts/ifcfg-eth0
- Deixando-a  assim.

DEVICE=eth0
MASTER=placa-backup   //Diz que a placa mestre é a placa-backup, herdara as configurações dela
SLAVE=yes             //Essa placa é a principal
BOOTPROTO=none
ONBOOT=yes

- Em seguida validar se a placa backup está habilitada para usar o modulo bonding. Criar o arquivo /etc/modprobe.d/bonding.conf e acresscentar :
alias		placa-backup 		bonding

## Para configurar o linux como um roteador:
- Adicionar o valor 1 ao arquivo cat /proc/sys/net/ipv4/ip_forward
echo 1 > /proc/sys/net/ipv4/ip_forward
- Para manter permanente a alteração, modificar o arquivo /etc/sysctl.conf
vi /etc/sysctl.conf 
net.ipv4.ip = 1

## Para configurar o linux como um dhcp
- Copiar o arquivo dhcp-*simple que fica dentro de /usr/share/doc/dhcp*/ para /etc/dhcp/
cp -a /usr/share/doc/dhcp*/dhcp-*simple /etc/dhcp/
- Alterar os endereços do arquivo.


## Para configurar o linux como um servidor dns.
- Instalar os pacotes bind , bind-utils e bind-libs.
- Alterar os arquivos /etc/named.conf
- O mapeamento do endereço ip para hostname é feito em arquivos de zonas localizadas no diretório /var/named


## Para configurar um linux como servidor proxy
- Instalar o squid
- Editar o arquivo /etc/squid/squid.conf
